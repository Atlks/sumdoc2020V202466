Rest url 接口跟俄们常用的编程api接口区别很大的....

Url接口的url 相等api黑头的method
参数数据json格式等于api黑头的 list/map/域对象
Url接口的返回码等于api黑头的异常处理...

所以,最佳实践是添加一个adapter...进程转换
两个映射法::一一映射and 抽象映射...
一一映射的过程是:::    url>>>>api方法名称.. 这个是使用最多的方法...
优点是rest url文档也能使用了...方便....
缺点是,,url api 设计的不好的时候儿, api映射的也继承了缺点...

.抽象映射提升了抽象级,不在一样映射...当url接口设计的不太好的时间,,这个很好使用...
